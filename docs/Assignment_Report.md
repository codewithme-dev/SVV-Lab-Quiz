# Assignment Report

## Section A – Validation Checklist Engineering
Stakeholder expectations were transformed into measurable validation criteria for secure login, transaction accuracy, session timeout, mobile responsiveness, and audit logging.

## Section B – Continuous Validation Evidence in CI Pipelines
GitHub Actions is used to run validation tests automatically, generate reports, and upload artifacts as evidence.

## Section C – Security-Oriented Validation and Structural Vulnerability Examination
OWASP ZAP is used to identify structural vulnerabilities such as SQL injection, XSS, and broken authentication.

## Section D – Validation Risk Structuring and Compliance Evidence Compilation
A risk register and compliance evidence dossier are maintained for audit readiness.
